# java-lotto-precourse
## 구현 예정 기능

- [x] 입출력 구현
    - "구입금액을 입력해 주세요." 문구 출력 및 로또 구입 금액 입력
    - "당첨 번호를 입력해 주세요." 문구 출력 및 당첨 번호 (쉼표 구분) 입력
    - "보너스 번호를 입력해 주세요." 문구 출력 및 보너스 번호 입력
    - camp.nextstep.edu.missionutils.Console 의 readLine() 을 사용하여 입력을 받는다.
    - 발행한 로또 수량 ("N개를 구매했습니다.") 출력
    - 발행한 로또 번호 목록 (오름차순 정렬) 출력
    - "당첨 통계" 및 등수별 당첨 내역 ("3개 일치 (5,000원) - 1개") 출력
    - 최종 수익률 ("총 수익률은 X.X%입니다.") 출력 (소수점 둘째 자리에서 반올림)

- [ ] 로또 구매 및 발급
    - 입력받은 구입 금액을 1,000원으로 나누어 구매할 로또 수량을 계산한다.
    - 구입 금액이 1,000원 단위가 아닐 경우 IllegalArgumentException 을 발생시킨다.
    - 구매 수량만큼 camp.nextstep.edu.missionutils.Randoms.pickUniqueNumbersInRange(1, 45, 6)를 호출하여 로또를 발급한다.
    - 발급받은 번호로 Lotto 객체를 생성하여 리스트에 저장한다.

- [ ] Lotto 클래스 구현
    - 제공된 Lotto 클래스를 활용한다.
    - 생성자에서 validate 메서드를 호출한다.
    - validate : 번호 개수가 6개가 아닐 경우 IllegalArgumentException 발생
    - validate : 로또 번호가 1~45 범위를 벗어날 경우 IllegalArgumentException 발생
    - validate : 로또 번호에 중복된 숫자가 있을 경우 IllegalArgumentException 발생

- [ ] 당첨 번호 및 보너스 번호 처리
    - 입력받은 당첨 번호(쉼표 구분 문자열)를 List<Integer> 로 파싱한다.
    - 당첨 번호 유효성 검사 (6개, 1~45 범위, 중복 없음) 후 Lotto 객체 또는 별도 클래스로 관리한다.
    - 입력받은 보너스 번호를 int 로 파싱한다.
    - 보너스 번호 유효성 검사(1~45 범위, 당첨 번호와 중복 없음)

- [ ] 당첨 통계 및 수익률 계산
    - 구매한 각 Lotto 객체를 당첨 번호와 비교하여 일치하는 번호 개수를 센다.
    - 5개 일치 시, 보너스 번호 일치 여부를 추가로 확인한다.
    - 당첨 기준(Enum 활용)에 따라 1등부터 5등까지 각 등수별 당첨 횟수를 집계한다.
    - 총 당첨금을 계산한다.
    - (총 당첨금 / 구입 금액) * 100을 기준으로 수익률을 계산하며, 소수점 둘째 자리에서 반올림

- [ ] 예외 처리
    - 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException 을 발생시킨다.
    - 예외 발생 시 "[ERROR]" 로 시작하는 에러 메시지를 출력한다.
    - 예외가 발생한 부분의 입력을 다시 받도록 구현한다.
    - System.exit()을 호출하지 않는다.

- [ ] 테스트 추가
    - JUnit 5와 AssertJ를 이용하여 각 기능에 대한 테스트 코드를 작성한다.
    - Lotto 클래스의 유효성 검사(개수, 범위, 중복) 로직을 테스트한다.
    - 구입 금액 예외 처리 테스트
    - 당첨 번호 및 보너스 번호 유효성 검사(범위, 중복) 테스트
    - 당첨 등수 판별 로직 (3~6개 일치, 보너스 일치) 테스트
    - 수익률 계산(반올림 포함) 테스트
    - UI 관련 로직(Console)은 테스트에서 제외한다.
